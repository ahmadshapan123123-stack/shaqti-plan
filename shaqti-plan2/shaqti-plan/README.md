# shaqti-plan

منصة تخطيط مشاريع عربية تُشغل Vite مع TypeScript وتستفيد من Konva، DnD Kit، Zustand، وTailwind مع دعم RTL من الوهلة الأولى.

## المميزات

- واجهة تخطيط تجمع بين لوحة المهام، كانفاس رسم متطور، ومحرر أرضي متعدد الأوضاع (رسم غرف، تحديد، إضافة جدران).
- مكتبة أثاث تفاعلية، مع سحب وإفلات ومواضع قابلة للتعديل والتكرار والتدوير عبر قائمة سياق أو الضغط على `R`.
- رفع صور أثاث مخصصة مع إزالة خلفية Remove.bg، وتخزين القطعة مباشرة في المكتبة بصيغة Base64.
- حفظ الغرف، الموقع، وإعدادات تأمين API في `localStorage` لإعادة إنشاء الحالة بعد إعادة التحميل.
- إعدادات منفصلة تتيح إدخال مفتاح Remove.bg لتشغيل المعالجة التلقائية داخل UploadFurnitureModal.

## تشغيل المشروع

```bash
npm install
npm run dev
```

### أوامر إضافية

- `npm run build`
- `npm run preview`

## مكتبة الأثاث

- `FurnitureLibrary` تظهر في مودال قابل للتصفية والبحث حسب التصنيف الأربع (صالة، نوم، مطبخ، حمام) وتُتيح السحب المباشر أو الوضْع بنقرة واحدة.
- تُدار القطع عبر Zustand (`useFurnitureStore`) التي تحفظ المكتبة والقطع المرسومة، وتضبط الاختيار عبر `selectedFurnitureId`.
- قائمة السياق تعتمد على اللغة العربية وتُتيح حذف، تكرار أو تدوير كل قطعة.

## إضافة أثاث مخصص

- `UploadFurnitureModal` عبارة عن نموذج مكون من ثلاث خطوات: رفع الصورة → معالجة الخلفية → إدخال الاسم/الأبعاد/التصنيف.
- يتم استدعاء API `https://api.remove.bg/v1.0/removebg` باستخدام المفتاح المخزّن في صفحة الإعدادات (`/settings`). عند الفشل يُسمح بالمتابعة بالصورة الأصلية.
- تحفظ القطعة الجديدة في `useFurnitureStore` مع الصورة المعالجة (base64)، وتظل متاحة بعد إغلاق التطبيق عبر `localStorage`.

## بنية Tailwind

- `tailwind.config.ts` يعرّف المصادر (`src/**/*.{ts,tsx}`) ويدمج `tailwindcss-rtl` مع خط `Cairo`.
- `postcss.config.cjs` يُشغّل Tailwind وAutoprefixer.
- `src/index.css` يفعّل RTL ويضبط الخلفية والخطوط العامة.
